# MapBot Reforged - Global AI Governance Protocols

## 1. Project Identity
* **Project Name:** MapBot Reforged
* **Core Technology:** Minecraft NeoForge 1.21.1 (Java 21)
* **Architecture:** Microservices pattern. Game Logic (NeoForge) <-> WebSocket <-> Bot Logic (NapCat/OneBot v11).
* **Primary Objective:** Rebuild the legacy Bukkit MapBot functionality using modern modding standards.

## 2. File System Authority (STRICT ENFORCEMENT)
* 🔴 **`./MapBotV4/`**: **RESTRICTED ZONE (READ-ONLY).**
    * Contains: Legacy source code.
    * Action: Reference ONLY. Never modify.
* 🟢 **`./MapBot_Reforged/`**: **ACTIVE WORKSPACE.**
    * Contains: New Source Code (NeoForge).
    * Action: Create, Edit, Refactor allowed.
* 🔵 **`./Project_Docs/`**: **KNOWLEDGE BASE.**
    * Contains: Documentation, Agreements, Reports.
    * Action: Read/Write.

## 3. Coding Standards
* **Language:** Java 21 (Preview features enabled if necessary).
* **Documentation:** All Javadoc and inline comments must be in **Simplified Chinese**.
* **Naming Convention:** CamelCase for classes/methods; SCREAMING_SNAKE_CASE for constants.
* **Safety:** All external I/O (WebSocket, File Ops) must be wrapped in `try-catch` blocks with meaningful error logging.

## 4. Operational Workflow
1.  **Read:** Before writing code, consult `./Project_Docs/Architecture` or `./MapBotV4` (for logic reference).
2.  **Plan:** State your intended changes in the chat.
3.  **Execute:** Modify files in `./MapBot_Reforged`.
4.  **Report:** After every task, append a log entry to `./Project_Docs/Reports/`.

## 5. Interaction Guidelines
* Respond to the user in **Simplified Chinese**.
* If a user request conflicts with these rules, cite this file and refuse.
