## 任务目标

升级基础签到功能，支持随机奖池抽奖、离线玩家奖励暂存（通过兑换码机制）以及游戏内兑换指令。

---

## 核心变更

### 1. 奖池配置 (`LootConfig`)
*   **路径**: `config/mapbot_loot.json`
*   **功能**: 支持基于权重的随机抽奖。
*   **默认组**:
    *   `Common (60%)`: 苹果、面包、煤炭等。
    *   `Rare (30%)`: 铁锭、金锭、粘液球。
    *   `Epic (9%)`: 钻石、绿宝石。
    *   `Legendary (1%)`: 下界合金、不死图腾。
*   **特色**: 每种稀有度都有专属的回复文案（如 "👑 天选之子！"）。

### 2. 签到状态机 (`SignManager`)
*   **Pre-roll 机制**: 玩家在输入 `#sign` 时即刻确定奖品。
*   **暂存流程**: 抽奖结果暂时保存在内存中，等待领取。
*   **在线领取**: 输入 `#accept` 直接发到背包。
*   **离线兑换**: 输入 `#cdk` 生成 4 位随机码（如 `MP-A1B2`），有效期 24 小时。

### 3. 命令体系升级
| 命令 | 场景 | 功能 |
| :--- | :--- | :--- |
| `#sign` | QQ | 执行抽奖，根据在线状态引导后续操作 |
| `#accept` | QQ | (仅在线) 确认领取暂存奖励 |
| `#cdk` | QQ | (仅离线) 将暂存奖励转换为兑换码 |
| `/mapbot cdk <code>` | 游戏 | 使用兑换码提取奖励 |

### 4. 游戏内指令注册
*   在 `MapBot.java` 中注册了 `/mapbot` 命令树。
*   实现了兑换码校验（检查是否过期、是否属于该玩家）。

## 交互流程示例

**场景 A: 玩家在线**
1.  输入 `#sign`
2.  Bot: "获得物品: [Epic] 钻石 x2。检测到您在线，请输入 #accept 领取"
3.  输入 `#accept`
4.  Bot: "发放成功！" (游戏内收到钻石)

**场景 B: 玩家离线**
1.  输入 `#sign`
2.  Bot: "获得物品: [Rare] 铁锭 x5。检测到您离线，请输入 #cdk 获取兑换码"
3.  输入 `#cdk`
4.  Bot: "您的兑换码: MP-X9Z1。请进服输入 /mapbot cdk MP-X9Z1"
5.  (玩家上线) -> 输入指令 -> 获得铁锭。
