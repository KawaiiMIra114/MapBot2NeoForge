## 1. 治理 KPI 定义

| KPI-ID | 指标名称 | 计算方式 | 数据源 | 目标值 | 超标动作 |
|---|---|---|---|---|---|
| KPI-01 | 巡检及时率 | 按时完成的巡检数 / 触发的巡检总数 | Evidence 目录 | ≥ 90% | 补跑巡检 + 记录债务 |
| KPI-02 | 阻断命中率 | 阻断了真实问题的次数 / 总阻断次数 | gate*.exit | ≥ 70% | 审查门禁规则有效性 |
| KPI-03 | 回滚演练达标率 | 演练通过次数 / 演练总次数 | 演练报告 | ≥ 80% | 修复演练失败项 |
| KPI-04 | RCA 完成率 | 72h 内完成 RCA 的事故数 / 事故总数 | 事故报告 | 100% | 升级处理 |
| KPI-05 | 合同漂移修复率 | 按时修复的漂移数 / 发现的漂移总数 | 巡检报告 | ≥ 85% | 升级为 P0 |
| KPI-06 | 差距清零进度 | 已关闭 Gap 数 / 总 Gap 数 | CURRENT_STATE | 逐版本递减 | 专项清零冲刺 |

## 2. 报告触发与节奏

| 报告类型 | 触发条件 | 产出时限 | 模板 |
|---|---|---|---|
| 事件报告 | EVT-03 (事故) | 24h 内 | §3.1 |
| 版本报告 | EVT-04 (发布前) | 发布前 48h | §3.2 |
| 月度报告 | EVT-06 (每月) | 月初 7 天内 | §3.3 |
| 专项报告 | KPI 超标 | 超标后 48h | §3.4 |

## 3. 报告模板

### 3.1 事件报告模板
```
# 事件报告 - [事件ID]
- 触发时间:
- 事件类型: EVT-0x
- 影响范围:
- 执行动作:
- 结论: PASS / 需跟进
- 证据路径:
- 后续行动:
```

### 3.2 版本报告模板
```
# 版本报告 - v[X.Y.Z]
- 门禁结果: gate01~gate11
- 编译状态: Alpha / Reforged
- 兼容审计: PASS / 违规项
- 演练结果: PASS / 失败项
- Go/No-Go: GO / NO-GO
```

### 3.3 月度报告模板
```
# 月度治理报告 - [YYYY-MM]
- KPI 汇总: (表格)
- 差距变化: +N / -N / 净值
- 巡检执行: N/M 完成
- 风险趋势: 上升 / 平稳 / 下降
- 改进行动: (列表)
```

### 3.4 超标报告模板
```
# KPI 超标报告 - [KPI-ID]
- 指标: [名称]
- 目标值: [X%]
- 实际值: [Y%]
- 偏差原因:
- 修复计划:
- 复检时间:
```

## 4. 可持续性保障
- KPI 数据来源均为已有 Evidence/Artifacts 文件
- 计算方式可脚本化（无需手动统计）
- 单人可在 30 分钟内完成月度报告
