## 1. 故障演练常态机制

### 1.1 演练触发条件

| 触发ID | 条件 | 演练类型 | 频率 |
|---|---|---|---|
| DRL-01 | 发布前 (EVT-04) | 全量回滚演练 | 每次发布 |
| DRL-02 | 关键链路改造后 | 链路故障注入 | 改造完成时 |
| DRL-03 | 事故复盘后 (EVT-03) | 防再发验证演练 | RCA 完成后 |
| DRL-04 | 依赖升级后 (EVT-05) | 兼容性回退演练 | 升级完成时 |

### 1.2 最小覆盖场景

| # | 场景 | 验证目标 | 通过标准 |
|---|---|---|---|
| S-01 | Bridge 连接超时 | 自动重连 + 消息不丢失 | 重连成功 + 消息补偿 |
| S-02 | Bridge 断连 | 优雅降级 + 本地缓存 | 服务不中断 |
| S-03 | 版本回滚 | 数据兼容 + 功能恢复 | 回滚后冒烟测试通过 |
| S-04 | 数据补偿 | CAS 冲突 + 幂等重试 | 数据最终一致 |

### 1.3 演练失败升级路径
1. 演练失败 → 记录 Failure Report
2. 分析根因 → 创建 Fix Action (P0)
3. 修复后 → 复验演练
4. 连续 2 次失败 → 阻断发布

## 2. 版本兼容与弃用审计

### 2.1 审计触发条件
- EVT-05: MC/NeoForge/JDA 版本变更
- EVT-02: 协议字段变更
- EVT-06: 定期审计 (每月)

### 2.2 兼容矩阵

| 维度 | 审计内容 | 审计方法 |
|---|---|---|
| 协议版本 | Bridge protocol_version | 双向版本协商检查 |
| 特性开关 | Feature Flag 状态 | Flag 清单对账 |
| 命令语义 | 命令参数/返回格式 | 命令签名回归测试 |
| 错误码 | 错误码兼容性 | 错误码映射表对账 |
| 依赖版本 | MC/NeoForge/JDA | build.gradle 版本锁定 |

### 2.3 弃用窗口管理

| 阶段 | 时长 | 动作 |
|---|---|---|
| Deprecated | 1 个版本周期 | 标记弃用 + 日志警告 |
| Sunset | 下一版本 | 移除代码 + 迁移指南 |
| Removed | 发布后 | 清理文档引用 |

### 2.4 兼容违规处理

| 违规级别 | 处理动作 |
|---|---|
| 破坏性变更 (无迁移路径) | 禁止发布 |
| 破坏性变更 (有迁移路径) | 延期发布 + 迁移指南 |
| 弃用超期未移除 | P1 技术债务 |
| 依赖版本不兼容 | 回退依赖版本 |

## 3. 审计产出
- 兼容矩阵报告 (每次审计生成)
- 弃用清单 (持续更新)
- 风险趋势图 (按月统计)
