## 故障注入场景

### FI-01 断连注入
| 项目 | 值 |
|---|---|
| 注入方式 | 关闭 Bridge WebSocket 连接 |
| 注入点 | BridgeServer.channelInactive |
| 持续时间 | 30s / 60s / 300s |
| 预期行为 | 客户端感知断连 → 告警 → 自动重连 |
| 状态机断言 | CONNECTED → DISCONNECTED → RECONNECTING → CONNECTED |
| 终态收敛 | 重连成功, 会话恢复 |
| 阈值 | 重连成功率 ≥ 99%, 最大延迟 ≤ 60s |

### FI-02 乱序注入
| 项目 | 值 |
|---|---|
| 注入方式 | 延迟部分消息 0-5s 随机 |
| 注入点 | Bridge 消息队列 |
| 影响范围 | 命令响应/事件通知 |
| 预期行为 | 基于 sequence/requestId 重排 or 丢弃过期 |
| 状态机断言 | 无重复处理, 无遗漏 |
| 终态收敛 | 所有消息最终处理或超时丢弃 |
| 阈值 | 重复处理数 = 0, 遗漏率 ≤ 1% |

### FI-03 超时注入
| 项目 | 值 |
|---|---|
| 注入方式 | Thread.sleep(10000) at 处理入口 |
| 注入点 | InboundHandler.handle / BridgeProxy |
| 影响范围 | 命令响应 |
| 预期行为 | 调用方收到超时响应, 无数据损坏 |
| 状态机断言 | PENDING → TIMEOUT → COMPENSATED |
| 终态收敛 | 数据一致, 无悬挂操作 |
| 阈值 | 超时响应率 100%, 数据一致性 100% |

### FI-04 重放注入
| 项目 | 值 |
|---|---|
| 注入方式 | 重发相同 requestId 消息 |
| 注入点 | Bridge 入站通道 |
| 影响范围 | 绑定/解绑/跨服等有副作用操作 |
| 预期行为 | 幂等保护: 仅首次执行, 后续忽略 |
| 状态机断言 | 副作用计数 = 1 |
| 终态收敛 | DB 状态与首次执行一致 |
| 阈值 | 重复副作用数 = 0 |

### FI-05 配置损坏注入
| 项目 | 值 |
|---|---|
| 注入方式 | 写入非法 JSON 到 config |
| 注入点 | AlphaConfig 配置文件 |
| 影响范围 | #reload 流程 |
| 预期行为 | 拒绝加载 → 保持旧配置 → 告警 |
| 状态机断言 | 配置版本不变 |
| 终态收敛 | 系统稳定运行在旧配置 |
| 阈值 | 配置版本不变, 系统可用 |

## FAILURE_MODEL 对照
| 模型状态 | FI 覆盖 | 验证 |
|---|---|---|
| PENDING → COMMITTED | FI-03 超时 → 补偿 | ✅ |
| PENDING → COMPENSATED | FI-01 断连 → 重连 | ✅ |
| 重试幂等 | FI-04 重放保护 | ✅ |
| 乱序容忍 | FI-02 消息重排 | ✅ |
| 配置韧性 | FI-05 损坏回退 | ✅ |

## 通过标准
1. 状态迁移可审计且终态收敛率 ≥ 99%
2. 重复副作用数 = 0
3. 所有注入后系统可自恢复

## 失败判据
1. PENDING 超期滞留未处理
2. 同 requestId 产生重复副作用
3. 注入后系统不可恢复
