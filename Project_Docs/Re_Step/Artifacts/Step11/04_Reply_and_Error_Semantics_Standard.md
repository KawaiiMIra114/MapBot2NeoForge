## 回执分类

### 成功回执
```
[成功] {具体操作结果}
示例: [签到成功] 获得 100 金币!
```

### 失败回执
```
[错误] {失败原因}
示例: [错误] 你还没有绑定游戏账号,请先使用 #id <游戏名> 绑定
```

### 超时回执
```
[超时] {目标服务} 响应超时,请稍后重试
示例: [超时] 服务器响应超时,请稍后重试
```

### 待确认回执
```
[等待] {等待说明}
示例: [等待] 解绑申请已提交,等待管理员审批
```

## 错误码映射

### Bridge 错误码 (已有)
| 错误码 | 含义 | 用户可见消息 |
|---|---|---|
| BRG-AUTH-101 | 认证失败 | 服务器连接认证失败 |
| BRG-AUTH-102 | Token过期 | 会话已过期 |
| BRG-VALIDATION-201 | 协议校验失败 | 请求格式错误 |
| BRG-TIMEOUT-301 | 响应超时 | 服务器响应超时 |
| BRG-EXEC-401 | 服务端执行失败 | 命令执行出错 |

### 命令层错误码 (E1 新增标准)
| 错误码 | 含义 | 用户可见消息 |
|---|---|---|
| CMD-PERM-001 | 权限不足 | 你没有执行此命令的权限 |
| CMD-ARGS-002 | 参数错误 | 用法: #{cmd} {usage} |
| CMD-BIND-003 | 未绑定 | 请先使用 #id <游戏名> 绑定 |
| CMD-STATE-004 | 状态冲突 | 操作无法在当前状态执行 |
| CMD-COOL-005 | 冷却中 | 请等待 {n} 秒后再试 |
| CMD-TARGET-006 | 目标不存在 | 找不到指定的玩家/服务器 |
| CMD-OFFLINE-007 | 目标离线 | 指定服务器当前离线 |

## 跨入口一致性保证

### QQ 命令入口
```
用户: #status
Bot: [服务器状态] 在线 3/20 | TPS 19.8 | ...
```

### API 控制面入口
```json
{
  "success": true,
  "error_code": null,
  "data": { "online": 3, "max": 20, "tps": 19.8 }
}
```

### 语义一致性规则
1. **同命令同结果**: QQ 和 API 对同一请求返回相同业务数据
2. **差异仅在表现层**: QQ 返回文本格式, API 返回 JSON 格式
3. **错误码映射**: 内部错误码 → QQ 中文消息 / API JSON error_code
