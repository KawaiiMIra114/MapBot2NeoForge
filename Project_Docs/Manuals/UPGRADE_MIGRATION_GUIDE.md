# UPGRADE_MIGRATION_GUIDE

## 1. 目标与范围
- 目标：让升级过程可预测、可验证、可回滚，避免“发布即线上调试”。
- 范围：MapBot 版本升级、依赖升级、配置结构升级、数据结构迁移。

## 2. 角色与职责
| 角色 | 职责 |
|---|---|
| Upgrade Owner | 制定计划与最终 Go/No-Go |
| Executor | 执行升级与回滚动作 |
| Verifier | 独立验证功能与指标 |
| Recorder | 记录步骤、证据与差异 |

## 3. 升级前校验（强制）
### 3.1 变更冻结与输入完整性
- 冻结窗口内禁止插入未评审变更。
- 确认输入包完整：版本说明、配置变更、数据迁移说明、回滚条件。

### 3.2 基线快照
- 版本快照：当前 tag/commit、构建产物 checksum。
- 配置快照：生产配置导出 + 密钥引用清单。
- 数据快照：关键数据备份并验证可恢复性。

### 3.3 预演与兼容性
- 在预发环境进行一次完整演练（含回滚演练）。
- 执行兼容性检查：协议、命令、权限矩阵、外部依赖 API。
- 通过标准：核心用例通过率 100%，非核心用例通过率 >= 95%。

## 4. 升级中校验（强制）
### 4.1 分阶段执行
1. 执行变更前健康检查（实例、依赖、告警状态）。
2. 灰度发布（建议 10% -> 30% -> 100%）。
3. 每阶段必须完成验收后再进入下一阶段。

### 4.2 运行时验证项
| 检查项 | 成功标准 | 失败处理 |
|---|---|---|
| 服务存活 | 无持续重启/无崩溃 | 立即暂停升级 |
| 错误率 | 不高于基线 + 20% | 回退到上一阶段 |
| 延迟 | P95 不高于基线 + 20% | 降级非核心功能 |
| 核心命令 | 全部可执行且权限正确 | 触发回滚评估 |
| 审计日志 | 关键动作可追溯 | 阻断继续发布 |

### 4.3 阻断条件（任一命中即停止）
- 数据迁移出现不可逆错误。
- 核心命令失败率 > 5% 且持续 10 分钟。
- 出现 Sev-0 / Sev-1 事故。

## 5. 升级后校验（强制）
### 5.1 功能验收
- 核心场景冒烟：登录绑定、消息桥接、管理命令、状态查询。
- 权限与安全：高危命令权限验证、审计字段完整性。

### 5.2 稳定性观察
- 观察窗口建议 >= 60 分钟。
- 对比基线：错误率、延迟、吞吐、告警数量。

### 5.3 文档与记录收口
- 更新版本记录与变更摘要。
- 归档升级执行记录、验收证据与问题清单。

## 6. 回滚策略与步骤（强制）
### 6.1 回滚触发条件
- 命中阻断条件。
- 升级后观察窗口内核心指标持续异常且无快速修复路径。

### 6.2 标准回滚流程
1. IC/Upgrade Owner 宣布回滚并冻结新操作。
2. 回退应用版本到上一稳定 tag。
3. 恢复配置快照（如有结构变更，执行反向迁移脚本）。
4. 恢复关键数据（仅在数据一致性受损时执行）。
5. 执行回滚后冒烟测试并恢复监控基线。

### 6.3 回滚验收
- 核心用例全部通过。
- 指标回到升级前可接受区间。
- 事故与回滚记录完整归档。

## 7. 验收标准
- 升级记录包含“前/中/后”三阶段证据。
- 回滚路径经过预演并可在 15 分钟内执行。
- 所有阻断条件与触发动作有明确责任人。

## 8. 记录与审计
| 记录项 | 必填字段 |
|---|---|
| 升级记录 | 版本、时间窗、执行人、验证人、结果 |
| 问题清单 | 现象、影响、根因、处置、状态 |
| 回滚记录 | 触发原因、步骤、耗时、恢复证据 |

## 10. 完整升级演练样例（前置快照 -> 灰度 -> 全量 -> 回滚）
说明：本样例用于演练，不要求真实故障。目标是验证“可升级、可观测、可回滚”三件事都成立。

### 10.1 前置快照（T-30min）
1. 记录当前稳定版本 `vA` 与目标版本 `vB`。
2. 导出配置快照、数据快照与监控基线（错误率、P95、告警数）。
3. 判定阈值（后续统一使用）：
   - 错误率阈值：`<= 基线 + 20%`
   - 延迟阈值：`P95 <= 基线 + 20%`
   - 核心命令成功率阈值：`>= 99%`
   - 阻断阈值：核心命令失败率 `> 5%` 持续 10 分钟

### 10.2 灰度发布（T+0 ~ T+20min）
1. 阶段 G1：10% 流量到 `vB`，观察 10 分钟。
2. 验收：错误率、P95、核心命令成功率满足阈值后进入 G2。
3. 阶段 G2：30% 流量到 `vB`，继续观察 10 分钟。
4. 判定：
   - 若任一指标超阈值，立即停止并回退到上一阶段。
   - 若命中阻断阈值，直接进入回滚流程。

### 10.3 全量发布（T+20 ~ T+80min）
1. 全量切换到 `vB`（100%）。
2. 执行核心冒烟：登录绑定、消息桥接、管理命令、状态查询。
3. 稳定性观察 60 分钟：
   - 通过：保持阈值内，且无 Sev-0/Sev-1。
   - 不通过：进入 10.4 回滚。

### 10.4 回滚演练（T+80min 触发）
1. 由 Upgrade Owner 宣布回滚，冻结新增操作。
2. 应用版本从 `vB` 回退到 `vA`。
3. 恢复配置快照（如配置结构变更则执行反向迁移）。
4. 若数据一致性异常，恢复关键数据快照并校验。
5. 回滚后验收（连续 30 分钟）：
   - 核心命令成功率 `>= 99%`
   - 错误率与 P95 回到基线 +20% 内
   - 关键告警恢复到升级前水平

### 10.5 演练结论记录
- 记录每个阶段的 Go/No-Go 结论、责任人、时间戳。
- 输出“是否可在 15 分钟内完成回滚”的实测结果。
- 未达标项必须进入行动项清单并指定 Owner 与截止日期。

## 11. 诊断分叉树：灰度成功但全量失败
触发条件：
- 10%/30% 灰度阶段通过，但 100% 全量后出现错误率或延迟超阈值。

诊断分叉树（按顺序执行）：
```text
Start: Full rollout fails, canary passed
|
|-- Q1: 失败是否与流量规模强相关？(并发提升后才出现)
|   |-- Yes -> Q1.1 资源瓶颈检查（CPU/内存/连接池/线程池）
|   |          |-- 命中 -> 结论A: 容量或限流参数不足，先降级/扩容
|   |          |-- 未命中 -> Q1.2 锁竞争/热点键检查
|   |-- No  -> Q2
|
|-- Q2: 是否仅在特定租户/群组/命令出现？
|   |-- Yes -> Q2.1 数据分布差异检查（脏数据/超大对象/冷分区）
|   |          |-- 命中 -> 结论B: 数据形态问题，执行数据修复或隔离
|   |          |-- 未命中 -> Q2.2 权限与配置差异检查
|   |-- No  -> Q3
|
|-- Q3: 外部依赖在全量阶段是否触发限额/超时？
|   |-- Yes -> 结论C: 外部依赖容量不足，启用缓存/熔断/重试退避
|   |-- No  -> Q4
|
|-- Q4: 灰度样本是否覆盖了真实高风险路径？
|   |-- No  -> 结论D: 灰度样本偏差，重设灰度分桶并复测
|   |-- Yes -> Q5
|
|-- Q5: 是否存在仅全量触发的异步任务/批处理/定时作业？
|   |-- Yes -> 结论E: 后台任务放大效应，错峰/限速后重试
|   |-- No  -> 结论F: 未定位，立即回滚并进入 Sev-1 排障流程
```

全量失败时的即时动作：
1. 先执行流量回退到上一个稳定阶段（100% -> 30% 或回滚 `vA`）。
2. 保留现场证据：指标快照、告警时间线、错误样本、依赖状态。
3. 按分叉树完成最短路径定位，不超过 15 分钟；超时直接回滚。
