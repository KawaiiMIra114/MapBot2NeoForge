# Security Policy

## 支持版本

| 版本 | 支持状态 |
|---|---|
| 最新发布 | ✅ 安全更新 |
| 上一个大版本 | ⚠️ 严重漏洞修复 |
| 更早版本 | ❌ 不支持 |

## 报告安全漏洞

**请勿通过公开 Issue 报告安全漏洞。**

请通过以下方式报告:
1. GitHub Security Advisory (推荐)
2. 发送邮件至项目维护者

### 报告内容

- 漏洞类型
- 受影响的组件/文件
- 复现步骤
- 影响评估
- 可能的修复方向 (如有)

## 响应 SLA

| 阶段 | 时间 |
|---|---|
| 确认收到 | 48 小时内 |
| 初步评估 | 72 小时内 |
| 修复发布 | 30 天内 (严重漏洞优先) |

## 披露策略

- 修复发布前不公开漏洞细节
- 修复后在 CHANGELOG 中标注安全修复
- 严重漏洞可能发布 CVE

## 安全最佳实践

- 不要在代码中硬编码 token/密钥
- 使用 `config/example.yml` 作为配置模板
- 生产配置文件已在 `.gitignore` 中排除
